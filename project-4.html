Ruche connectée Honeylink

Conception d'une ruche connectée utilisant le protocole LoRa pour surveiller à distance les données apicoles, avec une interface modulable sur ordinateur.

Project Overview

Dans le cadre de ma troisième année de BUT, j'ai conçu et mis en œuvre une solution IoT complète pour le suivi des ruches, combinant des capteurs embarqués, des communications longue portée et une infrastructure de traitement des données.

Le système repose sur une architecture robuste permettant de collecter en temps réel trois paramètres clés : le poids de la ruche (indicateur de production de miel et d'essaimage), ainsi que l'humidité et la température intérieure. Les données sont transmises toutes les 5 minutes via le protocole LoRa vers une gateway locale, qui les achemine ensuite vers la plateforme The Things Network (TTN).

Pour le traitement et la visualisation des données , j'ai développé une chaîne de traitement complète . Les informations remontées par TTN sont récupérées via l'API MQTT puis stockées dans une base de données Prometheus spécialisée dans les séries temporelles. Une instance Grafana connectée à cette base permet de générer des tableaux de bord analytiques. L'accès distant sécurisé à l'interface Grafana est assuré par un tunnel SSH via Tunnelmole, offrant ainsi aux apiculteurs un suivi en temps réel depuis n'importe quel appareil connecté.

Ce système a été conçu pour minimiser son impact environnemental, fonctionnant entièrement à l'énergie solaire. Sur le plan électromagnétique, les émissions LoRa (25 mW max, soit 14 dBm) sont particulièrement respectueuses des abeilles : la fréquence utilisée (868 MHz) est hors de leurs plages sensibles, la puissance est très faible (bien inférieure à un téléphone portable), et les émissions brèves (duty cycle < 1%) limitent l'exposition. Aucune étude sérieuse n'a montré d'effet nocif à cette puissance, d'autant que le système évite toute source de chaleur ou champ constant.

Ce projet m'a permis d'acquérir une expertise concrète sur l'ensemble de la chaîne IoT, depuis les capteurs embarqués jusqu'à la visualisation des données, en passant par les protocoles de communication longue distance et le traitement des séries temporelles . La solution développée répond précisément aux besoins du monde apicole tout en restant évolutive pour d'autres applications agricoles.
